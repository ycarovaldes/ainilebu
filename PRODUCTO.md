# PRODUCTO

## 1. Propuesta resumida

**Libreta Ainilebu** es una libreta clínica digital para una sola clínica
veterinaria. Reemplaza el cuaderno de papel manteniendo su mayor virtud —que es
inmediato y no exige estructura perfecta— pero agregando lo que el papel no puede:
búsqueda, historial por paciente, recordatorios automáticos (incluido pago) y
generación de documentos clínicos.

El producto se organiza en tres capas según prioridad:

1. **La libreta** (núcleo): registrar atenciones rápidas y buscar pacientes.
2. **Los recordatorios:** controles, vacunas, seguimientos y **pagos aplazados**.
3. **Lo especializado:** ficha de pacientes endócrinos y generación de documentos.

## 2. Para quién

- **Veterinaria** (usuaria principal): poco amigable con la tecnología. Prioriza
  mirar al paciente. Necesita registrar rápido y sin pasos obligatorios.
- **Recepción / colaboradores:** crean pacientes, ven recordatorios y pagos
  pendientes, ayudan con documentos.

## 3. Principios de diseño (no negociables)

- Menos campos es mejor. Menos clics es mejor.
- Autoguardado siempre; nada bloquea el flujo si falta información.
- Permitir texto libre + plantillas rápidas.
- Escribir primero, ordenar después.
- Interfaz limpia, grande y fácil de leer.
- Usable en tablet, notebook o PC de recepción con los mismos datos.
- Debe parecer una libreta ordenada, no un sistema complejo.

## 4. Criterios de éxito (medibles)

| Criterio | Meta |
|---|---|
| Registrar una consulta básica | < 60 segundos |
| Encontrar un paciente | < 5 segundos |
| Crear un paciente nuevo | 2 campos (nombre + teléfono) |
| Entender la pantalla | Sin capacitación extensa |
| Efecto sobre el papel | Lo reduce, no lo duplica |

Señales de mal diseño (si ocurren, está mal):
- Se siente más lento que la libreta.
- Exige demasiados campos obligatorios.
- Parece un ERP.

## 5. Propuesta de valor frente al papel

| El papel da… | Lo mantenemos |
|---|---|
| Inmediatez | Atención rápida con autoguardado |
| Escribir incompleto | Campos opcionales + estado "borrador" |
| Sin estructura perfecta | Texto libre + plantillas |
| Memoria del día | Dashboard del día |

| El papel falla en… | Lo resolvemos |
|---|---|
| No se puede buscar | Buscador < 5s |
| Se pierde info | Historial por paciente, en la nube |
| Sin recordatorios | Recordatorios automáticos (incl. pago) |
| Letra ilegible | Texto digital + dictado por voz |
| No se comparte | Acceso desde recepción y box |

## 6. Estrategia de adopción (usuaria reacia a la tecnología)

El riesgo número uno es que la veterinaria sienta que el sistema le quita tiempo.
La estrategia es que el sistema **nunca pida más de lo que el papel pedía**.

1. **Reemplazar solo el cuaderno primero.** Fase 1 = libreta digital. Nada más.
   No introducir todo de golpe.
2. **Paciente nuevo en 2 campos** (nombre + teléfono). Especie, raza, edad, etc.
   son opcionales y se pueden completar después o nunca.
3. **Dictado por voz** en las notas clínicas: mira al paciente, habla, el sistema
   escribe. Esta es la función que más reduce fricción para ella.
4. **Plantillas que rellenan casi todo:** un toque en "Consulta general" deja la
   atención casi lista; solo ajusta lo necesario.
5. **Autoguardado total:** no hay botón "Guardar" que recordar ni perder.
6. **Botones grandes, texto grande, una cosa por pantalla.**
7. **Modo borrador:** puede dejar atenciones a medio escribir y cerrarlas luego.
8. **Migración gradual:** los pacientes se crean solos a medida que llegan. No hay
   que cargar el cuaderno completo de una vez.
9. **Tolerancia a internet inestable:** borradores locales para que nunca se sienta
   más lento que el papel.
10. **Acompañamiento humano:** una sesión guiada de ~1 hora + una tarjeta impresa
    de "inicio rápido" pegada en el box (3 pasos: buscar → atención rápida → listo).
